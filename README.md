# CARLA + ROS2 Control Simulation Platform

This repository provides a high-performance simulation environment using **CARLA 0.9.16** and **ROS2 Humble**, containerized with Docker to ensure GPU acceleration and cross-platform consistency. The platform supports hybrid control modes: Automated (PID), Keyboard, and Hardware (Fanatec DD Pro).

---

## 🛠️ Prerequisites

* **OS:** Linux (Ubuntu 20.04/22.04 recommended).
* **Docker & Docker Compose**.
* **NVIDIA Container Toolkit:** Required for GPU passthrough to containers.
* **Python 3.10+** (Local environment for script execution).
* **Carla's minimum requirements:** check in the official page [here](https://carla.readthedocs.io/en/latest/start_quickstart/)

---

## 🏗️ Infrastructure Setup

### 1. Docker Services (Server-Side)
The simulation infrastructure is split into two independent services defined in `docker-compose.yml`:

* **`carla_dev`**: Runs the CARLA 0.9.16 server with native ROS2 support.
* **`ros2_dev`**: A desktop-enabled ROS2 Humble environment for telemetry and bridge verification.

**Launch the infrastructure:**
```bash
docker compose up --build
```

***Note: The first build may take a long time as it compiles Python 3.10.19 inside the CARLA container to ensure API compatibility.***

### 2. Local Python Environment (Client-Side)
The control scripts run locally to interface with the Dockerized server. Setup your environment as follows:

```bash
# Create a virtual environment
python3 -m venv carla_env

# Activate the environment
source carla_env/bin/activate

# Install dependencies
pip install -r requirements.txt
```

---

## 🏎️ Running the Simulation
Follow these steps to initialize the environment and run the control test:

### Step 1: Execute the Control Script
Run the test script from your local machine (ensure the Docker containers from the previous section are already running):

```bash
python3 custom_path_test.py
```

### Step 2: Verify ROS2 Data Stream
To inspect the topics generated by the simulation (Lidar, Odometry, IMU, etc.), access the ROS2 container:

```bash
# Enter the ROS2 container
docker exec -it ros2_tms /bin/bash

# List active ROS2 topics
ros2 topic list
```

## 🎮 Hybrid Control Interface
The `custom_path_test.py` script allows real-time switching between three distinct driving modes:

|Control Mode | Input | Activation |
| --- | --- |--- |
| Autonomous | PID Controller | Press 'P' (Keyboard) or Btn 101 (triangle in wheel △) |
| Manual (Keyboard) | 'WASD' Keys | Press Btn  98 (square in wheel ▢)|
| Manual (Cockpit) | Fanatec DD Pro | Press 'O' (Keyboard) |
| --- | --- |--- |

### Key Bindings:
* `C`: (Keyboard) / Btn 99 (x in wheel Ⓧ): Toggle Camera (Follow View / Cockpit View).
* `Q`: Toggle Reverse gear (Keyboard) / Btn 100 (circle in wheel ◯).
* `I`: Toggle On-Screen Display (HUD) for telemetry.

## 📊 Telemetry & HUD
The system renders a native Pygame HUD and "Oscilloscope" style graphs to track performance:

### General info
* **Vel:** Velocity
* **Steer:** Steering
* **E_Lon:** Longitudinal error
* **E_Lat:** Lateral error
* **Modo:** Mode (auto, manual)
* **Modo manual:** Manual mode (keyboard, wheel)
* **Camera:** Camera type

### Graphs
* Velocity Tracking: Real-time speed vs. Setpoint (25 km/h).
* Cross-Track Error: Real-time lateral distance from the reference trajectory.
* Control States: Feedback on steering angle, throttle, and brake pressure.
